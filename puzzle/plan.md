Es soll ein puzzle-spiel entwickelt werden
- ein klassisches puzzle wie es mit den ravensburger Kartons mit 1000 Teilen bekannt ist
- es soll ein festes set von motiven geben, bitte suche mal nach frein motiven Schloss, Berglandschaft, Fluslandschaft
- der user soll auch ein eigenes motiv hochladen können was im browser cache gehalten wird
- die anzahl puzzle teile soll wählbar sein 50, 100, 500, 1000
- das foto soll dann entsprechend per algorithmus zerlegt werden in die puzzle-teile

Oberfläche
- links werden 50 teile zur auswahl angezeigt, wie in realität etwas durcheinander hingeschüttet
- mit einem button "andere Teile" wird eine neue charge auf den tisch geworfen
- rechts daneben gibt es den Zielbereich, wo die teile eingepasst werden sollen
- die teile werden per maus oder finger in den rechten bereich gezogen, wennzweiteile zusammenpassen soll eine kleine
  animation das verdeutlichen
